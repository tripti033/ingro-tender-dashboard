  INGRO TENDER DASHBOARD — MANUAL COMMANDS
═══════════════════════════════════════════════════════════════════

  Always run from the project root:
    cd ~/work/ingro-tender-dashboard


───────────────────────────────────────────────────────────────────
  1. DEV SERVER
───────────────────────────────────────────────────────────────────

  # Start the Next.js app locally (http://localhost:3000)
  npm run dev

  # Production build (for sanity-check before deploy)
  npm run build

  # Typecheck only (no build)
  npx tsc --noEmit


───────────────────────────────────────────────────────────────────
  2. MAIN SCRAPER (runs automatically every 3h via GitHub Actions)
───────────────────────────────────────────────────────────────────

  # Scrape all sources (SECI, NTPC, GUVNL, MSEDCL, etc.)
  node scraper/index.js


───────────────────────────────────────────────────────────────────
  3. BID WINNER TRACKER (Gemini API + Google Search grounding)
───────────────────────────────────────────────────────────────────

  # Check all closed tenders for winners
  node scraper/result-tracker.js

  # Check one specific tender by NIT
  node scraper/result-tracker.js SECI-2025-TN000015

  # Try a different Gemini model (free-tier daily caps: 20/day each)
  GEMINI_MODEL=gemini-2.5-flash      node scraper/result-tracker.js
  GEMINI_MODEL=gemini-2.5-flash-lite node scraper/result-tracker.js  # default

  # Run without Google Search grounding (uses model's training data only)
  USE_GROUNDING=false node scraper/result-tracker.js

  Tip: if you hit "429 quota exceeded", switch to the other model
  (each model has its own 20/day bucket).


───────────────────────────────────────────────────────────────────
  3b. HISTORICAL BACKFILL (Gemini + Google Search)
───────────────────────────────────────────────────────────────────

  # Find BESS tenders floated during a date range (one Gemini call per month)
  node scraper/backfill-tenders.js 2023-06 2023-09        # Jun–Sep 2023
  node scraper/backfill-tenders.js 2024-01 2024-12        # full 2024
  node scraper/backfill-tenders.js 2023-06 2023-06 --dry  # preview only, no write

  Tenders found are written with source=["backfill"] and tenderStatus
  inferred ("awarded" if Gemini returned a winner, else "closed").
  They show up on /archives since they're historical.

  Tip: always preview with --dry first. Gemini can return different
  numbers of results each call.


───────────────────────────────────────────────────────────────────
  3i. ORPHAN CORRIGENDA CLEANUP
───────────────────────────────────────────────────────────────────

  # Delete corrigenda whose parent tender is no longer in the
  # database (scraper now skips writing them on future runs, but
  # use this to clean out any that already leaked through).
  node scraper/remove-orphan-corrigenda.js --dry
  node scraper/remove-orphan-corrigenda.js


───────────────────────────────────────────────────────────────────
  3g. TENDERDETAIL PURGE (one-off, already done 2026-04)
───────────────────────────────────────────────────────────────────

  # Deletes tenders whose ONLY source is TenderDetail. For merged
  # tenders, strips "TenderDetail" from sources without deleting.
  node scraper/remove-tenderdetail.js --dry
  node scraper/remove-tenderdetail.js

  The TenderDetail source is also commented out in scraper/index.js
  so it won't re-populate on the next scrape.


───────────────────────────────────────────────────────────────────
  3f. CHECKLIST EXTRACTOR (Ollama reads RfP PDF -> checklist items)
───────────────────────────────────────────────────────────────────

  # Extract for every tender that has a PDF but no checklist yet
  node scraper/extract-checklist.js

  # Extract for a single tender (used by the UI's "Extract" button)
  node scraper/extract-checklist.js UJVNL-T-04-EE-Solar-2025-26

  # Force re-extraction even for tenders that already have a checklist
  node scraper/extract-checklist.js --all

  Interactive — shows extracted items grouped by envelope, press y to
  write them, n to skip, a to auto-approve remaining, q to quit.


───────────────────────────────────────────────────────────────────
  3h. SEED COMPARABLE TENDERS (19 historical BESS benchmarks)
───────────────────────────────────────────────────────────────────

  # One-off: loads 19 past awarded BESS tenders (APTRANSCO, GUVNL,
  # NHPC, SECI etc.) with winning tariffs into Firestore so the
  # "Comparable Tenders" card on each tender detail page has a pool
  # to match against. Safe to re-run.
  node scraper/seed-comparables.js

  These are stored under NITs like BENCH-* and filtered out of
  /archives so they don't clutter your active-work views.


───────────────────────────────────────────────────────────────────
  3e. SEED DEMO TENDER (UJVNL Dhakrani, hand-verified fields)
───────────────────────────────────────────────────────────────────

  # Re-seed the Dhakrani 30MW/75MWh BESS tender (safe to re-run)
  node scraper/seed-dhakrani.js

  Viewable at: /tender/UJVNL-T-04-EE-Solar-2025-26


───────────────────────────────────────────────────────────────────
  3c. CORRIGENDUM DIFFS (LLM extracts what changed)
───────────────────────────────────────────────────────────────────

  # Diff every corrigendum vs its parent (writes summary + changes)
  node scraper/diff-corrigenda.js

  # Re-diff even corrigenda already processed (after prompt tweaks)
  node scraper/diff-corrigenda.js --retry

  # Only diff corrigenda under a specific parent NIT
  node scraper/diff-corrigenda.js SECI-2025-TN000011


───────────────────────────────────────────────────────────────────
  3d. FAKE CONTACT CLEANUP
───────────────────────────────────────────────────────────────────

  # Null out obvious placeholder contact fields (abc@..., 1234567890,
  # "John Doe" etc.) from all existing tenders. Pure pattern match,
  # no PDF re-fetch.
  node scraper/clean-fake-contacts.js --dry   # preview
  node scraper/clean-fake-contacts.js         # apply


───────────────────────────────────────────────────────────────────
  4. COMPANY MERGE SUGGESTIONS
───────────────────────────────────────────────────────────────────

  # Scan companies, write duplicate groups to /companies/merges
  node scraper/suggest-merges.js

  # Clear all pending suggestions first, then regenerate
  node scraper/suggest-merges.js --clear

  Review and approve/reject at: http://localhost:3000/companies/merges
  (or prod: <your-vercel-url>/companies/merges)


───────────────────────────────────────────────────────────────────
  5. LLM TOOLS (require Ollama running locally)
───────────────────────────────────────────────────────────────────

  # Start Ollama (if not already running)
  ollama serve

  # Interactive review of LLM extractions before they go to Firestore
  node scraper/llm-review.js                    # all tenders with gaps
  node scraper/llm-review.js --pdf              # also extract from PDFs
  node scraper/llm-review.js --retry-failed     # retry tenders LLM gave up on
  node scraper/llm-review.js SECI-2025-TN000015 # a specific tender

  # Note: tenders where LLM extracts nothing get llmExtractionFailed=true
  # and are skipped on future runs. They retry automatically only with
  # --retry-failed, or clear the flag once extraction succeeds.

  # Process news alerts for relevance score + categories
  node scraper/llm-alerts.js           # only un-scored alerts
  node scraper/llm-alerts.js --all     # re-process everything


───────────────────────────────────────────────────────────────────
  6. PDF ENRICHMENT
───────────────────────────────────────────────────────────────────

  # Download PDFs, extract fields (bid dates, EMD, etc.)
  node scraper/pdf-parser.js                    # all tenders
  node scraper/pdf-parser.js SECI-2025-TN000015 # one tender


───────────────────────────────────────────────────────────────────
  7. BULK IMPORTS (one-time / rare)
───────────────────────────────────────────────────────────────────

  # Import tenders from the Excel file
  node scraper/import-excel.js

  # Import CRM (companies + contacts + bids) from Excel
  node scraper/import-crm.js


───────────────────────────────────────────────────────────────────
  8. GIT / DEPLOY
───────────────────────────────────────────────────────────────────

  # See what changed
  git status
  git diff

  # Commit + push (Vercel auto-deploys on push to main)
  git add <files>
  git commit -m "your message"
  git push

  # Manually trigger the GitHub Actions scrape
  gh workflow run scrape.yml


───────────────────────────────────────────────────────────────────
  9. ENVIRONMENT VARIABLES (.env in project root)
───────────────────────────────────────────────────────────────────

  Required for scraping + writes:
    NEXT_PUBLIC_FIREBASE_API_KEY
    NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN
    NEXT_PUBLIC_FIREBASE_PROJECT_ID
    NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID
    NEXT_PUBLIC_FIREBASE_APP_ID
    FIREBASE_SCRAPER_EMAIL       (scraper@ingroenergy.com)
    FIREBASE_SCRAPER_PASSWORD

  Required for result-tracker:
    GEMINI_API_KEY

  Optional overrides:
    GEMINI_MODEL       (default: gemini-2.5-flash-lite)
    USE_GROUNDING      (default: true — set to "false" to disable)


───────────────────────────────────────────────────────────────────
  TYPICAL WORKFLOWS
───────────────────────────────────────────────────────────────────

  A) "Check who won the recently closed tenders":
       node scraper/result-tracker.js

  B) "Clean up duplicate companies":
       node scraper/suggest-merges.js
       → then review at /companies/merges

  C) "Force a fresh scrape without waiting for cron":
       node scraper/index.js
       node scraper/suggest-merges.js

  D) "Re-check a single tender":
       node scraper/result-tracker.js <NIT-NUMBER>
       node scraper/pdf-parser.js     <NIT-NUMBER>
